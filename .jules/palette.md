## 2025-02-13 - Keyboard Accessibility for Astro Overlays
**Learning:** When building custom overlays (like mobile menus) with vanilla JS in Astro `<script>` tags, toggling classes is not enough for accessibility. Toggling `hidden` hides visual elements but lacks semantic connection. The trigger must explicitly use `aria-controls` to link to the overlay ID. Additionally, standard keyboard accessibility requires the `Escape` key to securely close the overlay and restore focus to the original trigger button (`button?.focus()`) so keyboard/screen reader users don't lose their place in the DOM.
**Action:** Always add `aria-controls="[overlay-id]"` to overlay toggles and implement `Escape` key listeners that restore focus to the trigger button when building vanilla JS interactive overlays in Astro components.
